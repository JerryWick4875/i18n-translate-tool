# 长期记忆管理 Skill

长期记忆存储在 `~/.memory/` 目录下的 Markdown 文件中。`index.md` 是所有记忆文件的索引入口。

## 记忆目录结构

```
~/.memory/
├── index.md          # 索引文件，列出所有记忆文件及其摘要
├── projects.md       # 项目相关记忆
├── preferences.md    # 用户偏好设置
├── patterns.md       # 常用模式和约定
└── <topic>.md        # 其他主题文件
```

## 加载记忆

当用户说"加载记忆"、"读取记忆"、"load memory"、"查看记忆" 时：

1. 读取 `~/.memory/index.md`（如果不存在则告知用户暂无记忆）
2. 根据当前对话上下文，读取 index.md 中列出的相关主题文件
3. 将记忆内容整合到当前对话中，告知用户已加载哪些记忆

## 保存记忆

当用户说"记住"、"保存到记忆"、"save memory"、"记录下来" 时：

1. 判断信息属于哪个主题（项目/偏好/模式/其他）
2. 确保 `~/.memory/` 目录存在（`mkdir -p ~/.memory/`）
3. 将信息追加或更新到对应的主题文件
4. 更新 `~/.memory/index.md`，保持索引准确

## 删除/遗忘记忆

当用户说"忘记"、"删除记忆"、"forget" 时：

1. 从对应主题文件中删除相关内容
2. 如果主题文件变为空，删除该文件
3. 更新 `~/.memory/index.md`

## index.md 格式

```markdown
# 记忆索引

最后更新：<date>

## 文件列表

| 文件 | 描述 | 最后更新 |
|------|------|----------|
| [projects.md](projects.md) | 项目相关记忆 | 2026-03-23 |
| [preferences.md](preferences.md) | 用户偏好 | 2026-03-23 |

## 快速摘要

### projects.md
- 项目A：...
- 项目B：...

### preferences.md
- 偏好1：...
```

## 主题文件格式

每个主题文件使用清晰的 Markdown 结构，按小节组织内容，便于查找和更新。避免重复，保持简洁。

## 注意事项

- 记忆文件路径使用 `~/.memory/`，确保跨会话持久化
- 不要保存临时的、会话特定的内容
- 信息有变化时，更新而非追加，避免冗余
- 保存后告知用户已保存到哪个文件
